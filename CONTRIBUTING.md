# Contributing to ProducerBuddy

Conventions for humans and agents working in this repo.

## Build & run

The Xcode project is generated by [XcodeGen](https://github.com/yonaskolb/XcodeGen) — it is **not** committed.

```bash
brew install xcodegen swiftlint   # one-time
xcodegen generate                 # creates ProducerBuddy.xcodeproj
open ProducerBuddy.xcodeproj
```

Build/test from the command line:

```bash
xcodegen generate
xcodebuild test -scheme ProducerBuddy \
  -destination 'platform=iOS Simulator,name=iPhone 16'
```

## Architecture

Layering is intentional — keep dependencies pointing one direction:

```
Views / Components   (SwiftUI)        ── may import Services, Models
        │
Services             (pure logic)     ── NO SwiftUI; unit-tested
        │
Models               (SwiftData)      ── persistence types + value enums
        │
Support              (cross-cutting)  ── AppLinks, ReleaseSurface, Haptics, A11yID
```

Rules:

1. **Domain logic stays UI-free.** `SequencingEngine`, `AudioAnalyzer`, and
   `WaveformGenerator` must not import SwiftUI/UIKit so they remain unit-testable.
   If something is hard to unit test, it's probably in the wrong layer.
2. **No business rules in `View.body`.** Compute in a helper/service.
3. **One source of truth per concern** — external URLs in `AppLinks`, surface
   gating in `ReleaseSurface`, UI-test ids in `A11yID`.

## Testing

- **Swift Testing** (`import Testing`, `@Test`, `#expect`). Add tests for new
  domain logic before wiring UI.
- Drive estimators with synthesized signals (see `AudioAnalyzerTests`,
  `WaveformTests`) rather than fixture audio.
- Heavy/audio-dependent paths that can't run headless are verified manually on a
  simulator/device — note that in the PR.

## Accessibility (release gate)

Every interactive control ships with:

- An `accessibilityLabel` if it is icon-only.
- An `accessibilityIdentifier` from `A11yID` if it's used in UI tests.
- A ≥44×44pt touch target.

Decorative images use `.accessibilityHidden(true)`. Respect
`@Environment(\.accessibilityReduceMotion)` for non-essential animation. Don't
rely on color alone — pair it with a label or SF Symbol.

## Style

- SwiftLint must pass (`swiftlint`). Config in `.swiftlint.yml`.
- Match surrounding code: comment the *why*, not the obvious *what*.
- Conventional, descriptive commit messages.

## Release surface

Build wide, ship narrow. Hide unfinished areas behind a flag in
`ReleaseSurface` rather than deleting code. CI/dogfood builds can reveal
everything with the `-enable_full_product_surface` launch argument.
