# MixStack ‚Äî agent rules

Recurring conventions so agents don't reintroduce fixed mistakes.

## Project
- The `.xcodeproj` is generated by XcodeGen and **gitignored** ‚Äî never commit it.
  After editing `project.yml`, run `xcodegen generate`.
- Build/test with XcodeBuildMCP (see `.cursor/mcp.json`) or:
  `xcodebuild test -scheme MixStack -destination 'platform=iOS Simulator,name=iPhone 17 Pro'`.

## Architecture
- Services (`SequencingEngine`, `AudioAnalyzer`, `WaveformGenerator`) must not
  import SwiftUI/UIKit ‚Äî keep them pure and unit-tested. See `specs/Architecture.md`.
- One source of truth: URLs ‚Üí `AppLinks`, surface gating ‚Üí `ReleaseSurface`,
  UI-test ids ‚Üí `A11yID`.

## Accessibility (release gate)
- Icon-only controls need an `accessibilityLabel`; decorative images use
  `.accessibilityHidden(true)`; respect Reduce Motion. See `specs/Accessibility.md`.
- `Tab { ‚Ä¶ }` content is `TabContent`, not a `View` ‚Äî apply view modifiers to
  the view *inside* the tab closure, not to `Tab` itself.

## Layout
- Distinguish iPad from large-phone landscape with the device **idiom**, not
  horizontal size class alone.

## Migrations
- Adding stored properties to `@Model`s relies on lightweight migration during
  pre-release dev; introduce a versioned schema before shipping data the user
  can't lose.
