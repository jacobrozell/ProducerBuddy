Accessibility Statement

Last updated: 16 June 2026

We want MixStack to be usable by everyone and aim for WCAG 2.1 AA.

What's in place

  • VoiceOver labels on icon-only controls (play, pause, skip, stop, settings).
  • The waveform scrubber is exposed as an adjustable element so VoiceOver users can scrub.
  • System text styles for Dynamic Type, and a Light/Dark/System theme choice in Settings.
  • Reduce Motion is respected for non-essential animation.
  • Status is conveyed with labels and SF Symbols, not color alone.
  • Automated contrast tests on brand tokens in CI.

Known gaps

Full manual VoiceOver audits, large-text (AXXXL) verification, and contrast evidence across every screen are still in progress. Audit notes live in accessibility/audits/ in the repository.

If you hit an accessibility barrier, please let us know via the support page so we can prioritize a fix.
